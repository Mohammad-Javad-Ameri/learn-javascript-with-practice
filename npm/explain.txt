semantic versioning

major minor patch

major version change when there is a breaking change and the old code will not work with the new code

minor version change when there is a new feature and the old code will work with the new code

patch (bug fix) version change when there is a bug fix and the old code will work with the new code

install older version of a package = npm install <package name>@<version>

dependencies vs devDependencies

install devDependencies = npm install <package name> --save-dev 

we can use short = npm install <package name> -D

local vs global 

local = npm install <package name>

global = npm install -g <package name>

install 2 or more packages at once = npm install <package name1> <package name2>

package.json vs package-lock.json

package.json = package name, version, description, author, repository, keywords, license, bugs, homepage, dependencies, devDependencies

package-lock.json = dependencies, devDependencies, scripts, and other metadata
